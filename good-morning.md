# Good Morning - Session Primer

## Core Instructions to Remember

### 1. Memory Management
- Always use `journal/` directory for session memory
- Create/update markdown files with dates (e.g., `journal/2025-07-28-session-notes.md`)
- Log key findings, decisions, problems solved, and context discovered
- Reference previous journal entries for continuity
- Update existing entries rather than creating duplicates

### 3. Key Principles
- Do what's asked; nothing more, nothing less
- NEVER create files unless absolutely necessary
- ALWAYS prefer editing existing files
- NEVER proactively create documentation unless requested
- Use TodoWrite tool for complex multi-step tasks
- Be concise (4 lines or less unless detail requested)
- Follow existing code conventions and patterns
- Check for existing libraries before assuming availability
